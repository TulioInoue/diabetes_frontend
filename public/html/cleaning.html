As we can saw on top, we see two major and one minor problems:

  • Major problems:

    1. The first one is that polyuria and polydipsia have high correlation between themselves. That is because people that have high tendencies to drink lots of water also have needs to go to the bathroom, which makes work with thoses features features separately impossible. Therefore, we are going to apply PCA in order to unite these parameters.

    2. Also, we see that this dataset have unfortunately interview to many woman with diabetes. Looking into the graph, we can conclude that almost all of them have the disease, which is bad for the model to understand. Since we know that gender for its own do not affect diabetes chances, we decided to drop this column.

  • Minor:

    1. If we are trying to create a machine learning model that could be used later, it is also better to drop irritability column, since it can be misunderstood very easy - giving to the user a different output depending of the day.

3) Preparing data¶

3.1) Preprocessing¶

Here, we are going to create preprocessing ColumnTransformer variable called pre_processing that it is responsible to handle all sort of transformations with our dataset:

  • pca: responsible to reunite and treated polyuria and polydipsia as one single feature, given its high correlation between themselves and target class;
  • standard_scaler: good to uniform standard deviation of age in order to match the same ranges of categorical columns;
In [9]:
dataset = dataset.drop(["gender", "irritability"])

pca = PCA(n_components = 1)
one_hot_encoder = OneHotEncoder()
standard_scaler = StandardScaler()

pre_processing = ColumnTransformer(
    transformers = [
        ("pca", pca, ["polyuria", "polydipsia"]),
        ("age_encoder", standard_scaler, ["age"]),
    ],
    remainder = 'passthrough'
)
